# 安全政策

## 支持的版本

| 版本 | 支持状态 |
|------|---------|
| 1.0.x | ✅ 支持 |

## 报告安全漏洞

如果您发现了安全漏洞，请**不要**通过公开的 Issue 报告。

请通过以下方式私下报告：

1. 发送邮件至项目维护者
2. 使用 GitHub 的 [安全漏洞报告功能](https://github.com/renbon-wu/dddd-gui/security/advisories/new)

### 报告内容

请包含以下信息：

- 漏洞描述
- 影响范围
- 复现步骤
- 可能的修复方案（如有）
- 您的联系方式

### 响应时间

- 确认收到报告：24 小时内
- 初步评估：3 个工作日内
- 修复时间：根据漏洞严重程度决定

## 安全最佳实践

使用本工具时，请遵循以下安全实践：

1. **仅扫描授权目标** - 确保您有权限扫描目标系统
2. **保护 API 密钥** - 不要将 API 密钥提交到版本控制
3. **定期更新** - 保持工具和依赖项更新
4. **审计日志** - 在敏感环境中启用审计日志
