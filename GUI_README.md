# dddd-gui 使用指南

## 概述

dddd-gui 是 dddd 漏洞扫描工具的图形化界面版本，基于 Wails 框架构建，提供直观易用的操作界面。

## 快速开始

### 运行方式

1. **Windows**：双击 `dddd-gui.exe` 运行
2. **macOS**：双击 `dddd-gui.app` 运行
3. **Linux**：执行 `./dddd-gui` 运行

### 首次运行

程序启动后会自动创建必要的配置文件目录结构：

```
dddd-gui/
├── config/
│   ├── finger.yaml      # 指纹数据库
│   ├── workflow.yaml    # 工作流配置（指纹→POC映射）
│   ├── pocs/            # POC文件目录
│   └── config.yaml      # 系统配置
└── result/              # 扫描结果输出目录
```

## 核心功能模块

### 1. 扫描任务

- **新建任务**：输入目标地址、端口范围、扫描类型
- **任务历史**：查看所有扫描任务记录
- **任务模板**：保存和加载常用扫描配置

### 2. 目标管理

- **导入目标**：支持文件导入（TXT/CSV）和手动输入
- **批量操作**：批量扫描、批量删除
- **目标分组**：按业务分类管理目标

### 3. 指纹管理

- **查看指纹**：浏览所有已配置的指纹规则
- **添加指纹**：创建新的指纹识别规则
- **编辑指纹**：修改现有指纹规则
- **删除指纹**：移除不需要的指纹
- **测试指纹**：验证指纹规则有效性

### 4. POC管理

- **查看POC**：浏览所有可用的POC文件
- **上传POC**：导入新的POC文件
- **测试POC**：验证POC有效性
- **工作流管理**：配置指纹与POC的映射关系

### 5. 结果分析

- **结果概览**：扫描统计信息和图表
- **结果列表**：详细漏洞列表
- **导出报告**：支持CSV/JSON格式导出
- **生成报告**：生成HTML/PDF报告

### 6. 系统设置

- **全局配置**：代理、线程数、超时时间
- **API配置**：Fofa、Shodan、Censys密钥
- **字典管理**：管理爆破字典
- **界面设置**：主题、语言、字体大小

## 文件路径说明

### POC文件位置

**内置POC**：`./config/pocs/`

所有Nuclei格式的YAML文件都放在此目录下，程序启动时会自动加载。

**添加新POC**：
1. 将POC文件（.yaml格式）复制到 `config/pocs/` 目录
2. 在工作流中配置该POC与指纹的映射

### 指纹数据库

**文件路径**：`./config/finger.yaml`

格式示例：
```yaml
APACHE-Solr:
  - 'body="Solr Admin" && body="/solr/"'
Fortinet-sslvpn:
  - 'body="fgt_lang" && body="/sslvpn/portal.html"'
```

支持的指纹规则：
- `header="xxx"` - 返回头包含xxx
- `header!="xxx"` - 返回头不包含xxx  
- `header~="xxx"` - 返回头匹配正则
- `body="xxx"` - 响应体包含xxx
- `body!="xxx"` - 响应体不包含xxx
- `body~="xxx"` - 响应体匹配正则
- `body=="xxx"` - 响应体等于xxx
- `server="xxx"` - Server头包含xxx
- `title="xxx"` - 标题包含xxx
- `cert="xxx"` - 证书包含xxx
- `port="xxx"` - 端口匹配
- `protocol="xxx"` - 协议匹配
- `path="xxx"` - 路径匹配
- `body_hash="xxx"` - 响应体hash
- `icon_hash="xxx"` - 图标hash
- `status="xxx"` - 状态码匹配
- `content_type="xxx"` - 内容类型匹配
- `banner="xxx"` - TCP banner包含xxx

### 工作流配置

**文件路径**：`./config/workflow.yaml`

格式示例：
```yaml
APACHE-Solr:
  type:
    - root
  pocs:
    - CVE-2017-12629
    - CVE-2019-0193

nginx:
  type:
    - root
  pocs:
    - Tags@nginx
```

**type字段说明**：
- `root`：在根路径执行POC（如 `http://host:port/`）
- `dir`：在每个目录层级执行POC
- `base`：在发现资产的基础路径执行POC

**POC匹配方式**：
- 直接文件名：`CVE-2017-12629`（会匹配 `CVE-2017-12629.yaml`）
- Tags匹配：`Tags@nginx`（匹配所有带nginx标签的POC）

### 字典文件

**文件路径**：`./config/dict/`

支持的字典类型：
- 子域名字典：`subdomain.txt`
- 端口字典：`port.txt`
- 目录字典：`dir.txt`

## 使用便捷性优化建议

### 1. 一键扫描

建议在首页添加常用扫描模板的快捷入口：
- 内网扫描（默认端口）
- 外网扫描（全端口）
- 仅指纹识别
- 完整漏洞扫描

### 2. 智能目标识别

自动识别输入的目标类型：
- IP地址：自动识别为单个目标
- CIDR网段：自动识别为网段扫描
- 域名：自动识别为网站扫描

### 3. 快捷键支持

添加常用操作的快捷键：
- `Ctrl+N`：新建任务
- `Ctrl+S`：保存配置
- `Ctrl+R`：刷新列表
- `Ctrl+D`：删除选中项

### 4. 批量操作优化

支持通过拖拽方式批量添加目标到分组。

### 5. 搜索过滤

在各列表页面添加搜索框，支持快速过滤。

### 6. 配置备份

支持一键导出/导入所有配置文件。

## 常见问题

### Q: POC上传后无法使用？

A: 需要在工作流中配置该POC与指纹的映射关系。

### Q: 扫描结果为空？

A: 请检查：
1. 目标是否可达
2. 指纹规则是否正确
3. 工作流是否正确配置

### Q: 如何更新POC和指纹？

A: 
1. POC：将新POC文件放入 `config/pocs/` 目录
2. 指纹：编辑 `config/finger.yaml` 文件
3. 工作流：编辑 `config/workflow.yaml` 文件

### Q: 配置文件保存在哪里？

A: 配置文件保存在程序运行目录下的 `config/` 文件夹中。

## 技术支持

如遇到问题，请检查：
1. 程序运行目录是否有读写权限
2. Nuclei是否已正确安装
3. 网络连接是否正常
